Technical Notes and Restrictions

General

Changes in this release:

  • Both the Sales Configurator & Sales Configurator Workshop have been modified to work with the Unicode character set. Data stored in a Unicode enabled Oracle database can be viewed regardless of the input language and script used to enter the data. In other words, the Sales Configurator & Workshop are true Unicode applications.
  • Oracle's ODBC driver is now used to connect to the database, since Microsoft's ODBC driver (which was previously used) does not handle Unicode. The login dialog in the Workshop has been changed to only display data sources using Oracle's driver. The version of the driver must be at least 10.2.x, otherwise a warning message will appear when connecting to the database.
  • The entire code base is upgraded to Visual Studio .NET 2003.
  • Building the installation project no longer requires InstallShield Professional 6.3. The entire build process is managed from Visual Studio .NET 2003.
  • .NET framework is required to run the Workshop application.
  • The changes made for Unicode required that the old password system was re-written. This means that passwords used for profiles defined in the Sales Configurator Workshop must be re-entered and saved to disk.
  • The following tasks which were previously a part of the release, have been made obsolete and are a no longer required: _AgreementExtract, _CustInfoExtract, Reload Customer and _TermsExtract.

Regarding technical notes, restrictions and technical configuration, please refer to the Configuration Guide on the web.

Installation

This component contains three different parts:

  • IFS Sales Configurator Server- The Server Installation that installs tables, indexes, package, sequences and triggers in the Oracle database.
  • IFS Sales Configurator Workshop - This part installs the administrator client used to create Sales Rules. The workshop tool should be installed to run on an Oracle Database with IFS Applications installed. It needs to be installed on each machine where this functionality is required.
  • IFS Sales Configurator- This part installs both the background process that is used to validate Sales Rules for configurations within IFS Applications, as well as the Sales Configurator client. It needs to be installed on each machine where this functionality is required..

Setup programs for these tools are found in the Win32Client folder.

Note! It is important to do complete installations. Installing parts of the IFS Sales Configurator Server or installing the server without installing the IFS Sales Configurator will result in Oracle errors when using associated functionality in IFS Applications.

The installation instructions provided in the "getting started" user manuals do not correctly describe the installation process. When installing IFS Sales Configurator, always follow the instructions in these release notes instead.

The IFS Sales Configurator Server is built using the standard IFS/Configuration Builder (ConfigBuild.exe). The SalesConfiguratorWS.msi and SalesConfigurator.msi files may be either built in the IFS/Configuration Builder and deployed with Setup.exe (from Foundation1) or run manually.

Fresh Install remarks

Fresh Install of Sales Configurator - Server

The server part of the component can be installed either from the IFS Base Server Administrator or by creating an install template in the IFS Configuration Builder tool. The install template should be started from SQL*Plus to begin the installation. IFS R&D recommends use of SQL*Plus with the install template.

Regardless of which installation method is chosen, the following questions must be answered during the installation process.

    Component build options

    None.

Fresh Install of Sales Configurator

Installation on the necessary machines is done by performing the following steps:

  1. Install MS Internet Explorer 5.5 SP2 or higher (if not already present).

  2. Run SalesConfigurator.msi (either directly or through a Win32Client build) and follow the on-screen instructions. This will install the IFS XMS Bridge, Microsoft XML parser & SOAP and IFS Sales Configurator.

  3. If the application will be used from the IFS Applications web server to validate sales rules when configuring in the web client, you must manually specify the location of the Sales Configurator and install the XMS Socket Server. Check the release notes for the Configuration Characteristic Web (CFGCHW) component for details.

Fresh Install of Sales Configurator - Workshop

Installation is done by performing the following steps:

  1. Install MS Internet Explorer 5.5 SP2 or higher (if not already present)

  2. Install version 1.1 of the .NET framework.

  3. Create an ODBC data source that will connect you the IFS Applications Oracle database - see instructions below.

  4. Run SalesConfiguratorWS.msi (either directly or through a Win32Client build) and follow the on-screen instructions. This will install the IFS Sales Configurator Workshop.

    On some machines, an error message regarding a failed registration of COMCAT.DLL may appear. It is safe to ignore this message, since the DLL in these cases already have been registered.

  5. Installing Sales Configurator - Server would have created the SCONF_WORKSHOP role. Grant this SCONF_WORKSHOP role to all workshop users in the Oracle database as the application owner.

Creating an ODBC data source

  1. Start the Windows "ODBC Data Sources Administrator" from the Control Panel in Windows 9X or NT or from Start -> Administrative Tools -> Data Sources (ODBC) in Windows 2000 & Windows XP.
  2. If you want only this Windows user to run the IFS Sales Configurator, select the "User DSN" tab. If you want all users on the PC to have access, or if hosting this on a multi-user environment then select the "System DSN" tab.
  3. Click Add.
  4. Select the appropriate driver. Use Oracle's ODBC driver version 10.2 or higher.
  5. Enter the following configuration details:

    a) Specify the data source name (DSN). This is a name that you can choose to identify the DSN.
    b) Enter a description of the data source. Any text can be entered here.
    c) Do not specify a user name.
    d) In the "Server" field, enter the name of the Oracle SQL*Net or Net8 alias name for the database to which you wish to connect.

Upgrade Remarks

Note! This version only supports upgrades from version 8.1.0 or later. Upgrades from earlier versions must be done in two steps - an upgrade to version 8.1.0, followed by the upgrade to this version.

It is necessary to upgrade from an existing client installation. Previous versions of the Sales Configurator and Sales Configurator Workshop clients must be uninstalled before the new versions are installed. In addition, the SCONF_WORKSHOP role can simplify grants to users.

When upgrading from a previous release of the Sales Configurator, the data versions that were generated by the previous release of Sales Configurator Workshop must be re-generated. The Sales Configurator will present an error message if an attempt is made to open a data version created by a previous release of the Sales Configurator.

Getting Started

Verify that there are no invalid objects in the database.
Note! When using install template (created by IFS Configuration Builder), the installation ends with an automatic recompilation of all invalid objects.

Provided that installation has been performed according to the instructions above, it should be possible to start using both the Sales Configurator and Sales Configurator Workshop immediately. The Sales Configurator will be launched automatically as a background process from the standard IFS Applications clients when needed (e.g. when creating a Customer Order using a configurable sales part).

For more information on how to get going with the Sales Configurator Workshop, please refer to the "IFS Sales Configurator Workshop Getting Started" manual included in this release.

To use the Sales Configurator as a standalone application and transfer configurations into IFS Applications, IFS Connectivity must be configured to receive an ORDERS message. Refer to the documentation in IFS Applications. Also, the Internet Transaction Services (ITS) component must be installed.
